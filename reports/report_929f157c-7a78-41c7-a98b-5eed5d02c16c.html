Data Sources & Integration * **Primary Sources:** * **ERP System (e.g., SAP, Oracle, NetSuite):** * Supplier Master Data (Supplier Name, ID, Address, Business Unit association). * Purchase Orders (PO creation dates, promised delivery dates, total spend). * Goods Receipts (actual delivery dates, quantities). * Invoices (actual spend amounts). * **SRM / Supplier Risk Management System:** * Supplier Risk Scores. * Supplier Risk Classifications (High, Medium, Low). * Risk assessment dates. * Audit results, compliance data. * **Quality Management System (QMS):** * Quality inspection results. * Defect rates, rejection rates. * Supplier quality ratings. * **Internal Spreadsheets/Databases:** For any manual risk assessments or specific quality checks not integrated into main systems. * **Integration Method:** * **ETL/ELT Pipeline:** Utilize tools like Informatica, Talend, Azure Data Factory, AWS Glue, or custom Python scripts to extract, transform, and load data into a central Data Warehouse or Data Lakehouse. * **API Integrations:** For real-time or near real-time data from modern cloud-based systems. * **Direct Database Connections:** For on-premise ERP/QMS systems, though often less performant for large-scale reporting. ### 3. Data Preparation & Transformation This is a critical phase to ensure data quality, consistency, and usability for the dashboard. * **A. Core Supplier Master Data:** * **Unification:** Create a single, canonical `Supplier` dimension table by consolidating data from all source systems using `Supplier ID` as the primary key. * **Standardization:** Standardize `Supplier Name` (e.g., consistent casing, remove redundant suffixes). * **Business Unit Mapping:** Ensure each supplier is correctly mapped to one or more `Business Unit` dimensions. * **Deduplication:** Identify and merge duplicate supplier records. * **B. Supplier Risk Data:** * **Risk Classification Logic:** Define clear rules for mapping raw risk scores or criteria into `High`, `Medium`, `Low` categories. This might involve thresholds (e.g., Score > 80 = High, 50-80 = Medium, < 50 = Low). * **Risk Score Normalization:** If `Supplier Risk Score` comes from different systems with varying scales, normalize them to a consistent range (e.g., 0-100). * **Temporal Risk:** Store historical risk classifications and scores with an effective `Date` to enable trend analysis. * **C. Delivery Data:** * **On-Time Delivery Flag:** For each delivery record (from Goods Receipts), calculate a boolean/binary flag: * `IsOnTimeDelivery = 1` if `ActualDeliveryDate <= PromisedDeliveryDate`. * `IsOnTimeDelivery = 0` otherwise. * **Data Granularity:** Ensure delivery data is at the individual shipment or line-item level to accurately count deliveries. * **D. Quality Data:** * **Quality Rating Standardization:** If `Quality Rating Score` comes from various sources, standardize it to a common scale (e.g., 1-5 or 0-100). * **Aggregation:** If quality data is at a very granular level (e.g., per item inspected), aggregate it to a per-delivery or per-supplier-per-period level for dashboard purposes. * **E. Purchase Spend Data:** * **Aggregation:** Sum `Invoice Amount` or `Purchase Order Value` by `Supplier ID`, `Business Unit`, and `Time` period. * **Currency Conversion:** If operating in multiple currencies, convert all spend amounts to a single reporting currency using daily or monthly exchange rates. * **F. Time Dimension:** * Create a robust `Date` dimension table containing `Year`, `Quarter`, `Month`, `Week`, `Day`, `DayOfWeek`, `IsWeekend` etc. This will be crucial for all time-based filtering and trend analysis. * **G. Data Model:** * Implement a **Star Schema** in the Data Warehouse. * **Fact Tables:** * `Fact_SupplierPerformance`: Containing measures like `OnTimeDeliveryCount`, `TotalDeliveries`, `QualityRatingScore`, `PurchaseSpendAmount`, and foreign keys to dimensions. * `Fact_SupplierRisk`: Containing `SupplierRiskScore`, `SupplierRiskClassification`, and foreign keys to dimensions. * **Dimension Tables:** * `Dim_Supplier` (ID, Name, Attributes) * `Dim_BusinessUnit` (ID, Name) * `Dim_Time` (Date, Year, Quarter, Month, etc.) * `(Optional) Dim_RiskClassification` (ID, Description) ### 4.
